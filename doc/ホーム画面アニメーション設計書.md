# ホーム画面アニメーション設計書：休肝日つくーる

## 1. 概要

ホーム画面（`app/(tabs)/index.tsx`）のステータスカードに、ステータス変更時と2連続休肝日達成時のアニメーション・ハプティクスフィードバックを追加する。

ユーザーの操作に対する視覚的・触覚的フィードバックを提供し、達成感を演出する。

## 4. アニメーション仕様

### 4.1. ステータス変更アニメーション

**トリガー**: `todayStatus` が変更されたとき（前回値と異なる場合）

**動作**:
- カードが `1.0 → 0.97 → 1.0` にスケール変化（パルスアニメーション）
- Duration: 各 75ms（合計 150ms）
- `withSequence` + `withTiming` を使用

**ハプティクス**:
- `todayStatus === "kyukan"` の場合のみ `ImpactFeedbackStyle.Light` を発火
- Web環境ではスキップ（`Platform.OS !== "web"` ガード）

### 4.2. 2連続達成アニメーション

**トリガー**: `hasConsecutive` が `false → true` に変化したとき

**動作**:
- カード上に緑色（`#4CAF50`）のオーバーレイが `opacity 0 → 1 → 0` でフラッシュ
- Duration: 各 150ms（合計 300ms）
- `pointerEvents="none"` でタッチイベントを透過

**ハプティクス**:
- `NotificationFeedbackType.Success` を発火
- Web環境ではスキップ

## 5. 実装詳細

### 5.1. SharedValue

```typescript
const cardScale = useSharedValue(1);           // カードのスケール値
const cardOverlayOpacity = useSharedValue(0);  // 緑オーバーレイの不透明度
```

### 5.2. Ref（前回値の追跡）

```typescript
const prevStatusRef = useRef<DayStatus>(todayStatus);
const prevHasConsecutiveRef = useRef(hasConsecutive);
```

- `useRef` で前回値を追跡し、`useEffect` 内で変化を検出
- 初回レンダリング時にはアニメーションを発火しない

### 5.3. AnimatedStyle

```typescript
const cardAnimatedStyle = useAnimatedStyle(() => ({
  transform: [{ scale: cardScale.value }],
}));

const overlayAnimatedStyle = useAnimatedStyle(() => ({
  opacity: cardOverlayOpacity.value,
}));
```

### 5.4. JSX 変更

- `todayCard` の外側 `View` → `Animated.View` に変更
- `cardAnimatedStyle` を style に追加
- 内部に `Animated.View`（緑オーバーレイ）を `StyleSheet.absoluteFill` で追加

## 6. アニメーションガイドラインとの整合性

| ガイドライン | 実装 | 適合 |
| :--- | :--- | :--- |
| Duration: 80〜150ms | 75ms + 75ms = 150ms | ✅ |
| スケール変化: 0.95〜0.98 | 0.97 | ✅ |
| `withTiming` を優先 | `withTiming` のみ使用 | ✅ |
| `withSpring` は使わない | 使用していない | ✅ |
| Platform.OS ガード | Web環境でハプティクスをスキップ | ✅ |
