# オンボーディング設計書：休肝日つくーる

## 1. 概要

初回起動時に表示される3ステップのオンボーディング画面。アプリの紹介、目標設定、通知設定をガイドし、ユーザーがスムーズに利用を開始できるようにする。

各ステップはスキップ可能で、スキップした場合はデフォルト設定が適用される。

## 4. 画面構成

### 4.1. Stack.Screen 設定

```tsx
<Stack.Screen name="onboarding" options={{ presentation: "fullScreenModal", headerShown: false }} />
```

- `fullScreenModal` で全画面表示（バック遷移を防止）
- `router.replace("/onboarding")` で遷移するため、戻るジェスチャーなし

### 4.2. ステップ構成

```
ScreenContainer
└── View (container)
    ├── ステップインジケータ（3つのドット）
    │
    ├── Step 0: アプリ紹介
    │   ├── 絵文字 🍵
    │   ├── タイトル「休肝日を、無理なく続けよう」
    │   ├── 説明文
    │   ├── 「はじめる」ボタン → setStep(1)
    │   └── 「スキップ」リンク → completeOnboarding()
    │
    ├── Step 1: 目標設定
    │   ├── 絵文字 🎯
    │   ├── 週の目標休肝日数（ステッパー: − / 値 / +）
    │   ├── 2連続休肝日を目指す（Switch）
    │   ├── 「次へ」ボタン → setStep(2)
    │   └── 「スキップ」リンク → completeOnboarding()
    │
    └── Step 2: 通知設定
        ├── 絵文字 🔔
        ├── 夜間リマインダー（Switch）
        ├── 時刻選択チップ（18:00, 19:00, 20:00, 21:00, 22:00, 23:00）※Switch ON時のみ表示
        ├── 「通知を許可して完了する」/「完了する」ボタン → completeOnboarding()
        └── 「スキップ」リンク → completeOnboarding()
```

## 5. 動作仕様

### 5.1. OnboardingGate（`app/_layout.tsx`）

`AppProvider` 内部に配置されるレンダリングなし（`return null`）コンポーネント。

```tsx
function OnboardingGate() {
  const router = useRouter();
  useEffect(() => {
    AsyncStorage.getItem("onboarding_completed").then((val) => {
      if (!val) router.replace("/onboarding");
    });
  }, [router]);
  return null;
}
```

- アプリ起動時に `onboarding_completed` フラグを確認
- フラグがない場合、`router.replace("/onboarding")` でオンボーディング画面へ遷移
- `replace` を使うことでバックスタックに履歴を残さない

### 5.2. completeOnboarding 関数

完了時（最終ステップまたはスキップ）に以下を順番に実行:

1. `patchSettings()` で目標設定・通知設定を保存
2. `reminderEnabled && Platform.OS !== "web"` の場合、`requestNotificationPermission()` を呼ぶ
3. `AsyncStorage.setItem("onboarding_completed", "true")` で完了フラグを保存
4. `router.replace("/(tabs)")` でメイン画面へ遷移

### 5.3. 通知権限の二重リクエスト回避

- `requestNotificationPermission()` は内部で `notification_permission_requested` フラグを設定する
- `NotificationSetup` はこのフラグを確認してからリクエストする
- オンボーディングで権限リクエスト済みの場合、`NotificationSetup` は再リクエストしない

### 5.4. 2回目以降の起動

- `onboarding_completed` が `"true"` のため、`OnboardingGate` は何もしない
- 通常のタブ画面が即座に表示される

## 6. UI 実装方針

| UI 要素 | 実装 | 備考 |
| :--- | :--- | :--- |
| ステップインジケータ | `View` × 3（ドット）| アクティブ: `primary`、非アクティブ: `border` |
| ステッパー | `Pressable`（style prop）| 設定画面と同じパターン |
| Switch | React Native `Switch` | 設定画面と同じパターン |
| 時刻チップ | `Pressable`（style prop）| 選択中: `primary` 背景 + 白文字 |
| ボタン | `Pressable`（style prop）| **className 禁止ルール遵守** |
