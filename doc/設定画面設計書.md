# 設定画面設計書：休肝日つくーる

## 1. 概要

設定画面（Settings）は、ユーザーが目標休肝日数・通知・データ管理を行うための画面である。
5番目のタブとして追加され、ホーム画面ヘッダーの ⚙️ ボタンからも遷移できる。

---

## 2. 関連ファイル

| ファイル | 役割 |
| :--- | :--- |
| `lib/store.ts` | `AppSettings` 型定義、デフォルト値、永続化関数 |
| `lib/app-context.tsx` | Context に `settings` / `patchSettings` / `resetAllData` を提供 |

## 3. データ設計

### 3.1. AppSettings 型

```typescript
interface AppSettings {
  weeklyGoalDays: number;           // 週の目標休肝日数（1〜5、デフォルト: 2）
  requireConsecutive: boolean;      // 2連続休肝日を目指すか（デフォルト: true）
  reminderEnabled: boolean;         // 夜間リマインダー ON/OFF（デフォルト: true）
  reminderTime: string;             // リマインダー時刻 "HH:mm"（デフォルト: "20:00"）
  achievementNotification: boolean; // 達成通知 ON/OFF（デフォルト: true）
}
```

### 3.2. AsyncStorage キー

| キー | 内容 |
| :--- | :--- |
| `kyukoubi_settings_v1` | `AppSettings` の JSON 文字列 |

- 既存の `kyukoubi_store_v1`（DailyRecord）とは**独立したキー**で管理
- キーが存在しない場合は `defaultSettings()` のデフォルト値を使用

### 3.3. 永続化関数（store.ts）

| 関数 | 説明 |
| :--- | :--- |
| `defaultSettings()` | デフォルト値の `AppSettings` を返す |
| `loadSettings()` | AsyncStorage から設定を読み込み。未保存時はデフォルト値を返す |
| `saveSettings(settings)` | AsyncStorage に設定を保存 |
| `clearAllData()` | `kyukoubi_store_v1` と `kyukoubi_settings_v1` の両方を削除 |

### 3.4. Context API（app-context.tsx）

| フィールド / メソッド | 型 | 説明 |
| :--- | :--- | :--- |
| `settings` | `AppSettings` | 現在の設定値（useState で管理） |
| `patchSettings(patch)` | `(patch: Partial<AppSettings>) => Promise<void>` | 設定の部分更新（マージ → 保存 → state 更新） |
| `resetAllData()` | `() => Promise<void>` | 全データリセット（records と settings を初期化） |

## 4. 画面構成

### 4.1. タブ設定

- **タブ順序**: 5番目（ホーム / 週間計画 / 記録 / 振り返り / **設定**）
- **アイコン**: `gearshape.fill`（icon-symbol.tsx に既存のマッピング `"settings"` を使用）
- **タイトル**: `"設定"`

### 4.2. セクション構成

```
ScreenContainer
└── ScrollView
    ├── ヘッダー: "設定"
    │
    ├── セクション: 目標設定
    │   ├── 週の目標休肝日数（ステッパー: − / 値 / +）
    │   └── 2連続休肝日を目指す（Switch トグル）
    │
    ├── セクション: 通知設定
    │   ├── 夜間リマインダー（Switch トグル）
    │   ├── リマインダー時刻（トグル ON 時のみ表示、18:00〜23:00の6択から選択）
    │   ├── 達成通知（Switch トグル）
    │   └── ※ Phase 2 実装予定の注記テキスト
    │
    └── セクション: データ管理
        └── 全データリセットボタン（error カラー）
            └── 確認ダイアログ → resetAllData() 実行
```

### 4.3. UI 部品の実装方針

| UI 要素 | 実装 | 備考 |
| :--- | :--- | :--- |
| セクション見出し | `Text` + muted カラー + uppercase 風 | `fontSize: 11` |
| セクションカード | `View` + surface 背景 + border + borderRadius | `borderRadius: 16` |
| トグルスイッチ | React Native `Switch` | `trackColor` に `primary` / `border` を使用 |
| 日数ステッパー | `Pressable`（style prop）+ `−` / `+` ボタン | 範囲: 1〜5、範囲外で `disabled` |
| リマインダー時刻 | `Pressable` + 選択肢パネル | 18:00〜23:00（1時間刻み6択）をタップで選択 |
| リセットボタン | `Pressable`（style prop）+ error カラー | Web: `window.confirm()` / Native: `Alert.alert()` |

## 5. 動作仕様

### 5.1. 目標休肝日数ステッパー

- `−` ボタン: `weeklyGoalDays` を 1 減らす（最小値 1 で disable）
- `+` ボタン: `weeklyGoalDays` を 1 増やす（最大値 5 で disable）
- 変更時にハプティクス（Light）を発火

### 5.2. トグルスイッチ

- 切り替え時に `patchSettings()` で即座に保存
- ハプティクス（Medium）を発火
- `reminderEnabled` が OFF の場合、リマインダー時刻行は非表示

### 5.3. 全データリセット

- ボタン押下で確認ダイアログを表示
  - **Web**: `window.confirm()` を使用
  - **Native**: `Alert.alert()` を使用（destructive スタイル）
- 確認後 `resetAllData()` を実行
  - `records` → 空の `{ records: {} }` にリセット
  - `settings` → `defaultSettings()` にリセット
  - 全画面が自動で初期状態に戻る

## 6. 通知機能について

通知機能は実装済み。詳細は `doc/通知機能設計書.md` を参照。

- `expo-notifications` によるローカル通知（リマインダー・達成通知）
- リマインダー時刻は 18:00〜23:00 の6択から選択可能
- 設定変更時に `NotificationSetup`（`app/_layout.tsx`）が自動でスケジュール更新
