# 振り返り画面強化設計書：休肝日つくーる

## 1. 概要

振り返り画面（`app/(tabs)/review.tsx`）に以下の4つの新機能を追加し、ユーザーが自身の飲酒パターンをより深く理解し、モチベーションを維持できるようにする。

1. **連続達成バッジ** — 休肝日の実績に応じてバッジを自動解除・表示
2. **曜日別飲酒パターングラフ** — 過去4週間の曜日別平均飲酒量を棒グラフで可視化
3. **月次サマリーカード** — 月タブ選択時に休肝日数・先月比・達成率を表示
4. **月次レポート共有** — React Native の `Share` API でテキスト形式のレポートを共有

## 3. データ設計

### 3.1. AppStore 拡張

```typescript
export interface AppStore {
  records: Record<string, DailyRecord>;
  badges: string[]; // 解除済みバッジIDの配列
}
```

- `loadStore()` の初期値: `{ records: {}, badges: [] }`
- 既存データとの互換性: ロード時に `badges` プロパティがない場合は空配列をセット

## 4. 関数仕様

### 4.1. バッジ解除チェック

#### `checkNewBadges(records, existingBadges, weeklyGoalDays): string[]`

レコード更新のたびに呼ばれ、新たに解除されたバッジIDの配列を返す。既に解除済みのバッジは含まない。

| バッジID | 解除条件 |
| :--- | :--- |
| `first_kyukan` | いずれかのレコードに `status === "kyukan"` がある |
| `first_consecutive` | 日付順で2日連続 kyukan がある |
| `first_weekly_goal` | いずれかの週で kyukan 日数 >= `weeklyGoalDays` |
| `three_weeks_streak` | 3週連続で週間目標達成 |
| `one_month_streak` | 4週連続で週間目標達成 |

### 4.2. 曜日別平均計算

#### `computeWeekdayAverages(records): { dayIndex, label, avg }[]`

- 過去28日間のレコードを集計
- 曜日ごとの平均 `actualDrinks` を計算
- 返り値: 7要素の配列（dayIndex 0=月, 1=火, ..., 6=日）

### 4.3. 月次サマリー計算

#### `computeMonthlySummary(records, year, month, weeklyGoalDays): MonthlySummary`

| 返り値フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `kyukanDays` | `number` | 今月の休肝日数 |
| `prevMonthKyukanDays` | `number` | 先月の休肝日数 |
| `diff` | `number` | 先月比（正=増加、負=減少） |
| `achievementRate` | `number` | 達成率 %（上限100） |
| `comment` | `string` | 達成率に応じた4段階メッセージ |

達成率の計算: `kyukanDays / (月の週数 × weeklyGoalDays) × 100`

| 達成率 | コメント |
| :--- | :--- |
| 100% 以上 | 目標達成！この調子で続けましょう💪 |
| 75〜99% | あと少しで目標達成です！ |
| 50〜74% | 順調に進んでいます。もう少し頑張りましょう |
| 50% 未満 | まだ伸びしろがあります。少しずつ増やしていきましょう |

## 5. コンポーネント仕様

### 5.1. `BadgeList`（`components/BadgeList.tsx`）

横スクロールの FlatList で全バッジを表示するコンポーネント。

- **Props**: `earnedBadges: string[]`
- 解除済みバッジ: カラー表示（`primary` 背景の丸 + 絵文字 + 名前）
- 未解除バッジ: グレーアウト（opacity: 0.3、`border` 背景）
- `useMemo` でバッジデータをメモ化

### 5.2. `WeekdayChart`（`components/WeekdayChart.tsx`）

`react-native-svg` による曜日別平均飲酒量の棒グラフ。

- **Props**: `data: { dayIndex, label, avg }[]`
- `useWindowDimensions().width` でグラフ幅を動的に計算
- 最大値の曜日: `orange` カラー、他: `primary` カラー
- グラフ下にコメント: 「○曜日が最も飲みやすい傾向があります」
- `Animated.createAnimatedComponent(Svg)` は使わない（CLAUDE.md制約）

## 6. 画面レイアウト（`app/(tabs)/review.tsx`）

### 6.1. 配置順序（上から下）

```
┌─────────────────────────────┐
│ ヘッダー: 振り返り            │
├─────────────────────────────┤
│ バッジリスト（横スクロール）    │  ← 新規
├─────────────────────────────┤
│ タブ切替: 週 / 月             │
├─────────────────────────────┤
│ ← 期間ナビゲーション →        │
│                             │
│ KPI カード × 4               │
│                             │
│ 月次サマリーカード（月のみ）    │  ← 新規
│                             │
│ 杯数推移グラフ（既存）         │
│                             │
│ 曜日別平均グラフ              │  ← 新規
│                             │
│ 飲みたい理由ランキング（既存）  │
│                             │
│ インサイト（既存）             │
│                             │
│ [月次レポートを共有]（月のみ）  │  ← 新規
└─────────────────────────────┘
```

### 6.2. 月次サマリーカード

- `computeMonthlySummary` の結果を3カラムで表示:
  - 休肝日数 / 先月比（+N日/-N日、色分け） / 達成率
- 下部にコメントテキスト
- `surface` 背景の角丸カード

### 6.3. 月次レポート共有

- React Native 標準の `Share.share({ message })` でテキスト共有
- `expo-sharing` は不使用（テキスト共有にはファイルURL不要）

共有テキストフォーマット:

```
【休肝日つくーる 月次レポート】
2026年3月
休肝日: 8日（先月比 +3日）
達成率: 85%
目標達成！この調子で続けましょう💪
```

## 7. バッジ解除フロー（`lib/app-context.tsx`）

```
patchRecord(date, patch)
  ↓
updateRecord() → newStore を取得
  ↓
checkNewBadges(newStore.records, newStore.badges, settings.weeklyGoalDays)
  ↓
新規バッジあり?
  ↓ YES
newStore.badges に追加 → saveStore(newStore)
  ↓
setStore({ ...newStore })
  ↓
（既存の達成通知チェックが続く）
```

## 9. スタイリング注意事項

- **Pressable** には `style` prop を使用（`className` は `onPress` が発火しないバグあり）
- **FlatList** でバッジリストを表示（`ScrollView` + `.map()` は不使用）
- **useMemo** で `computeWeekdayAverages`、`computeMonthlySummary`、バッジデータをメモ化
- **色指定** は `useColors()` フックのテーマトークンを使用
- **react-native-svg** の Svg コンポーネントを `Animated.createAnimatedComponent()` でラップしない
