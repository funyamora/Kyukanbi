# 週間スワイプ操作設計書：休肝日つくーる

## 1. 概要

週間計画画面（`app/(tabs)/weekly.tsx`）の曜日行に、横スワイプによるステータス変更操作を追加する。

- **右スワイプ** → 休肝日に設定
- **左スワイプ** → 飲酒OKに設定
- **タップ** → 従来のサイクル切替（未定→休肝日→飲酒OK→未定…）

## 4. SwipeableDayRow コンポーネント

### 4.1. Props

| Prop | 型 | 説明 |
| :--- | :--- | :--- |
| `date` | `string` | 日付文字列（`"YYYY-MM-DD"`） |
| `isToday` | `boolean` | 今日の日付かどうか |
| `rec` | `DailyRecord` | その日の記録データ |
| `colors` | `ThemeColorPalette` | テーマカラー |
| `onSetKyukan` | `(date: string) => void` | 休肝日に設定するコールバック |
| `onSetOk` | `(date: string) => void` | 飲酒OKに設定するコールバック |
| `onTap` | `(date: string) => void` | タップ時のサイクル切替コールバック |

### 4.2. ジェスチャー構成

```
Gesture.Race(tap, pan)
```

- `Gesture.Tap()` — タップで `onTap(date)` を呼ぶ
- `Gesture.Pan()` — 横スワイプで `onSetKyukan` / `onSetOk` を呼ぶ
- `Gesture.Race()` — 先に認識された方が勝つ

### 4.3. Pan ジェスチャーの設定

| 設定 | 値 | 目的 |
| :--- | :--- | :--- |
| `activeOffsetX` | `[-10, 10]` | 水平 10px 以上で Pan を有効化 |
| `failOffsetY` | `[-10, 10]` | 垂直 10px が先行した場合は FlatList のスクロールに委譲 |

### 4.4. スワイプ閾値

| 定数 | 値 | 用途 |
| :--- | :--- | :--- |
| `SWIPE_THRESHOLD` | `60px` | この距離を超えるとステータス変更が確定 |
| `SWIPE_CLAMP` | `80px` | translateX の最大値（それ以上は動かない） |

---

## 5. アニメーション仕様

### 5.1. 行のスライド

- `translateX` SharedValue でスワイプに追従
- `±SWIPE_CLAMP (80px)` にクランプ
- スワイプ終了後に `withTiming(0, { duration: 120 })` で元の位置にスナップバック

### 5.2. 背景色プレビュー

スワイプ中に行の背後に色付きラベルが表示される:

| スワイプ方向 | 背景色 | ラベル |
| :--- | :--- | :--- |
| 右（→） | `#4CAF50`（緑） | 🍵 休肝日 |
| 左（←） | `#FF6B35`（オレンジ） | 🍺 飲酒OK |

- `interpolate` で `translateX: 0→SWIPE_THRESHOLD` を `opacity: 0→1` にマッピング
- `Extrapolation.CLAMP` で範囲外をクランプ

### 5.3. ハプティクス

| 操作 | フィードバック |
| :--- | :--- |
| タップ（サイクル切替） | `ImpactFeedbackStyle.Light` |
| スワイプ確定（休肝日/飲酒OK） | `ImpactFeedbackStyle.Medium` |

---

## 6. FlatList との共存

`failOffsetY([-10, 10])` により、ユーザーが先に垂直方向にスワイプした場合は Pan ジェスチャーが失敗し、FlatList の縦スクロールが通常通り機能する。

水平方向が先行した場合のみ Pan が有効化され、行のスライドが始まる。

## 7. コールバック

```typescript
const handleSetKyukan = useCallback(async (date: string) => {
  await patchRecord(date, { status: "kyukan" });
  // + haptics (Medium)
}, [patchRecord]);

const handleSetOk = useCallback(async (date: string) => {
  await patchRecord(date, { status: "ok" });
  // + haptics (Medium)
}, [patchRecord]);
```
