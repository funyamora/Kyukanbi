# 週間スワイプ操作設計書：休肝日つくーる

## 1. 概要

週間計画画面（`app/(tabs)/weekly.tsx`）の曜日行に、横スワイプによるステータス変更操作を追加する。

- **右スワイプ** → 休肝日に設定
- **左スワイプ** → 飲酒OKに設定
- **タップ** → 従来のサイクル切替（未定→休肝日→飲酒OK→未定…）

## 4. SwipeableDayRow コンポーネント

### 4.1. Props

| Prop | 型 | 説明 |
| :--- | :--- | :--- |
| `date` | `string` | 日付文字列（`"YYYY-MM-DD"`） |
| `isToday` | `boolean` | 今日の日付かどうか |
| `rec` | `DailyRecord` | その日の記録データ |
| `colors` | `ThemeColorPalette` | テーマカラー |
| `onSetKyukan` | `(date: string) => void` | 休肝日に設定するコールバック |
| `onSetOk` | `(date: string) => void` | 飲酒OKに設定するコールバック |
| `onTap` | `(date: string) => void` | タップ時のサイクル切替コールバック |

### 4.2. ジェスチャー構成

- **スワイプ**: `Gesture.Pan()` を `GestureDetector` で処理
- **タップ**: 通常の `Pressable`（`onPress`）で処理

> **注意**: 当初は `Gesture.Race(tap, pan)` でタップとスワイプを合成していたが、Web環境で `GestureDetector` の子要素のインラインスタイルが正しく更新されない問題があったため、タップは `Pressable` に分離した。

### 4.3. Pan ジェスチャーの設定

| 設定 | 値 | 目的 |
| :--- | :--- | :--- |
| `activeOffsetX` | `[-10, 10]` | 水平 10px 以上で Pan を有効化 |
| `failOffsetY` | `[-10, 10]` | 垂直 10px が先行した場合は FlatList のスクロールに委譲 |

### 4.4. スワイプ閾値

| 定数 | 値 | 用途 |
| :--- | :--- | :--- |
| `SWIPE_THRESHOLD` | `80px` | この距離を超えるとステータス変更が確定 |
| `SWIPE_CLAMP` | `120px` | translateX の最大値（背景ラベルが完全に見える余裕を確保） |

---

## 5. アニメーション仕様

### 5.1. 行のスライド

- `translateX` SharedValue でスワイプに追従
- `±SWIPE_CLAMP (120px)` にクランプ
- スワイプ終了後に `withTiming(0, { duration: 120 })` で元の位置にスナップバック

### 5.2. 背景色プレビュー

スワイプ中に行の背後に色付きラベルが表示される:

| スワイプ方向 | 背景色 | ラベル |
| :--- | :--- | :--- |
| 右（→） | `#4CAF50`（緑） | 🍵 休肝日 |
| 左（←） | `#FF6B35`（オレンジ） | 🍺 飲酒OK |

- `interpolate` で `translateX: 0→SWIPE_THRESHOLD` を `opacity: 0→1` にマッピング
- `Extrapolation.CLAMP` で範囲外をクランプ

### 5.3. ハプティクス

| 操作 | フィードバック |
| :--- | :--- |
| タップ（サイクル切替） | `ImpactFeedbackStyle.Light` |
| スワイプ確定（休肝日/飲酒OK） | `ImpactFeedbackStyle.Medium` |

---

## 6. FlatList との共存

`failOffsetY([-10, 10])` により、ユーザーが先に垂直方向にスワイプした場合は Pan ジェスチャーが失敗し、FlatList の縦スクロールが通常通り機能する。

水平方向が先行した場合のみ Pan が有効化され、行のスライドが始まる。

## 8. Web対応の注意点

### 8.1. インラインスタイルの回避

React Native Web では、`style={[staticStyle, { backgroundColor: dynamicValue }]}` のようなインラインスタイルオブジェクトの動的更新が正しく反映されないことがある（内部のスタイルキャッシュが古い値を保持する）。

**対策**: ステータスに応じた色は `StyleSheet.create` で事前定義し、ステータスをキーにして切り替える。

```tsx
// NG — インラインオブジェクト（Webで色が古いまま残る）
<View style={[styles.badge, { backgroundColor: conf.bg, borderColor: conf.border }]}>

// OK — 事前定義StyleSheet（CSSクラスベースで確実に切り替わる）
const STATUS_BADGE_STYLES = StyleSheet.create({
  kyukan:    { backgroundColor: "#E8F5E9", borderColor: "#4CAF50" },
  ok:        { backgroundColor: "#FFF3E0", borderColor: "#FF6B35" },
  undecided: { backgroundColor: "#F2F2F7", borderColor: "#E5E5EA" },
});
<View style={[styles.badge, STATUS_BADGE_STYLES[rec.status]]}>
```

### 8.2. GestureDetector とタップの分離

Web環境では `GestureDetector` が内部DOMをキャッシュし、子要素のスタイル更新を妨げることがある。タップ操作は `Pressable` で処理し、`GestureDetector` はスワイプ（Pan）専用にする。

### 8.3. スワイプ後のタップ防止（swipedRef）

Web環境では Pan の `onEnd` 発火後に `Pressable` の `onPress` も発火し、スワイプで設定したステータスが `handleToggle` のサイクル切替で上書きされる。`swipedRef`（`useRef<boolean>`）でスワイプ確定を記録し、直後の `onPress` を無視する。

```tsx
// Pan.onEnd 内でスワイプ確定時にフラグを立てる
runOnJS(markSwiped)();
runOnJS(onSetKyukan)(date);

// handlePress でフラグが立っていればタップを無視
if (swipedRef.current) {
  swipedRef.current = false;
  return;
}
```

### 8.4. 連打防止

`handleToggle` に `useRef` ベースの楽観的ロックを入れ、前回の `patchRecord` 完了前に次のタップが発火しても無視する。
