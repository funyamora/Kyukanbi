# 休肝日つくーる — Claude Code 開発ガイド

> このドキュメントは、Expoモックアプリをベースに Claude Code で本格開発を進めるための計画書・タスクリストです。

---

## 1. プロジェクト概要

「休肝日つくーる」は、飲酒習慣を持つ人が **週2日の連続休肝日** を無理なく実現するためのiOS/Androidネイティブアプリです。

| 項目 | 内容 |
| :--- | :--- |
| フレームワーク | Expo SDK 54 (React Native 0.81) |
| 言語 | TypeScript 5.9 |
| スタイリング | NativeWind v4 (Tailwind CSS) |
| 状態管理 | React Context + AsyncStorage |
| ナビゲーション | Expo Router v6 |
| テスト | Vitest |
| パッケージマネージャ | pnpm |

---

## 2. セットアップ手順

```bash
# 1. ZIPを解凍してディレクトリに移動
unzip kyukoubi-tsukuru.zip
cd kyukoubi-tsukuru

# 2. 依存関係のインストール
pnpm install

# 3. 開発サーバーの起動（Web）
pnpm dev:metro

# 4. iOSシミュレーターで起動
pnpm ios

# 5. Androidエミュレーターで起動
pnpm android

# 6. テストの実行
pnpm test
```

## 4. 開発フェーズ計画

モックアプリをベースに、以下の順序で本格開発を進めることを推奨します。

### Phase 1 — データ層の強化（優先度：高）

現在はモックデータ（AsyncStorage）で動作していますが、より堅牢なデータ管理が必要です。

| タスク | 概要 |
| :--- | :--- |
| 週間目標の設定機能 | 週の目標休肝日数（デフォルト2日）をユーザーが設定できるようにする |
| 設定画面の実装 | 目標設定・通知設定・データリセットを管理する設定画面を追加 |
| データ移行ロジック | アプリバージョンアップ時のAsyncStorageスキーマ移行処理 |

### Phase 2 — 通知機能（優先度：高）

習慣化に最も効果的な機能です。

| タスク | 概要 |
| :--- | :--- |
| 夜間リマインダー | 毎晩20時「今日の記録をつけましょう」通知 |
| 飲む前の一声通知 | 飲酒OK日の夕方に「上限を宣言しましたか？」通知 |
| 達成通知 | 2連続休肝日達成時の祝福通知 |

### Phase 3 — UX改善（優先度：中）

ユーザーの継続率を高める機能です。

| タスク | 概要 |
| :--- | :--- |
| オンボーディング | 初回起動時の目標設定フロー（3ステップ） |
| ウィジェット対応 | ホーム画面ウィジェットで今日のステータスを確認 |
| ハプティクス強化 | 達成時・警告時のフィードバック強化 |
| アニメーション | 画面遷移・状態変化のアニメーション追加 |

### Phase 4 — 分析・インサイト強化（優先度：中）

振り返り画面をより価値あるものにします。

| タスク | 概要 |
| :--- | :--- |
| 月次レポート | 月間サマリーをシェアできる画像として出力 |
| 連続達成バッジ | 週・月単位の達成バッジ表示 |
| 飲酒パターン分析 | 曜日・理由別の傾向グラフ追加 |

### Phase 5 — App Store申請準備（優先度：低）

| タスク | 概要 |
| :--- | :--- |
| プライバシーポリシー | アプリ内表示用のプライバシーポリシー画面 |
| App Store スクリーンショット | 申請用スクリーンショットの作成 |
| アクセシビリティ対応 | VoiceOver対応・Dynamic Type対応 |

## 6. 開発時の注意事項

### NativeWind の使い方

```tsx
// ✅ 正しい: className を使う
<View className="flex-1 bg-background p-4">
  <Text className="text-foreground text-lg font-bold">タイトル</Text>
</View>

// ❌ 注意: Pressable には className を使わない（onPress が発火しない場合あり）
// Pressable には必ず style prop を使う
<Pressable
  style={({ pressed }) => [styles.btn, pressed && { opacity: 0.8 }]}
  onPress={handlePress}
>
```

### カラートークン一覧

| トークン | ライト | ダーク | 用途 |
| :--- | :--- | :--- | :--- |
| `bg-background` | #F5F5F7 | #1C1C1E | 画面背景 |
| `bg-surface` | #FFFFFF | #2C2C2E | カード背景 |
| `text-foreground` | #1C1C1E | #F5F5F7 | 主要テキスト |
| `text-muted` | #8E8E93 | #AEAEB2 | 補助テキスト |
| `bg-primary` | #4A90D9 | #5BA3E8 | プライマリアクション |
| `bg-orange` | #FF6B35 | #FF8C5A | 飲酒関連アクション |
| `bg-purple` | #667EEA | #7B93F5 | 代替行動関連 |
| `text-success` | #4CAF50 | #66BB6A | 達成・成功 |
| `text-warning` | #F0A500 | #FBBF24 | 警告 |
| `text-error` | #EF4444 | #F87171 | エラー |

### データ構造

```typescript
// lib/store.ts より
export interface DailyRecord {
  date: string;           // "YYYY-MM-DD"
  status: "kyukan" | "ok" | "undecided";
  declaredLimit: number | null;   // 宣言した上限杯数
  drinkingReason: string | null;  // 飲みたい理由
  actualDrinks: number | null;    // 実際の杯数（0〜8、8は「8杯以上」）
  satisfaction: "great" | "okay" | "regret" | "toomuch" | null;
  memo: string;
  alternativeAction: string | null;
}

export interface AppStore {
  records: Record<string, DailyRecord>; // key: "YYYY-MM-DD"
}
```

### Context の使い方

```typescript
import { useAppStore } from "@/lib/app-context";

function MyComponent() {
  const { store, today, getRecord, patchRecord } = useAppStore();

  // 今日のレコードを取得
  const todayRecord = getRecord(today);

  // レコードを更新
  await patchRecord(today, { status: "kyukan" });
}
```

## 7. よくある問題と解決策

| 問題 | 原因 | 解決策 |
| :--- | :--- | :--- |
| Pressable の onPress が反応しない | NativeWind の className が干渉 | `style` prop を使う |
| 新しい画面が404になる | Expo Router のファイルベースルーティング | `app/` 配下に正しいパスでファイルを作成 |
| 新しいタブアイコンが表示されない | icon-symbol.tsx にマッピングがない | `MAPPING` オブジェクトに追加する |
| ダークモードで背景が白い | ScreenContainer を使っていない | `ScreenContainer` コンポーネントでラップする |
| AsyncStorage の変更が反映されない | Context の再レンダリングがない | `patchRecord` 後に `setStore` を呼ぶ（app-context.tsx 参照） |
| Webでステータス切替時に色が更新されない | React Native Web のインラインスタイルキャッシュ問題 | 動的な色は `StyleSheet.create` で事前定義し、キーで切り替える（インラインオブジェクト `{ backgroundColor: x }` を避ける） |
| Web で GestureDetector の子要素が更新されない | GestureDetector が内部DOMをキャッシュする | タップは `Pressable` で処理し、GestureDetector はスワイプ専用にする |
| Web でスワイプ後にタップが発火してステータスが上書きされる | Pan の onEnd 後に Pressable の onPress も発火する | `useRef` フラグでスワイプ確定を記録し、直後の onPress を無視する |
