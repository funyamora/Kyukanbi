# 通知機能設計書：休肝日つくーる

## 1. 概要

ローカル通知機能は、ユーザーの休肝日習慣を支援するために以下の2種類の通知を提供する。

1. **夜間リマインダー** — 毎日指定時刻に、その日のステータスに応じた通知を送信
2. **達成通知** — 2連続休肝日を達成した瞬間に即時通知を送信

バックエンドは不要で、すべて `expo-notifications` によるローカル通知で実現する。

## 4. 関数仕様

### 4.1. 権限管理

#### `requestNotificationPermission(): Promise<boolean>`

- Web環境では即座に `false` を返す
- `getPermissionsAsync()` で現在の権限を確認
- 未許可の場合 `requestPermissionsAsync()` でダイアログ表示
- Android では通知チャンネルを作成
- `notification_permission_requested` フラグを AsyncStorage に保存
- 権限が付与されたかどうかを返す

### 4.2. リマインダースケジューリング

#### `scheduleReminder(time: string, store: AppStore): Promise<void>`

- 既存のリマインダー通知をキャンセル
- `"HH:mm"` 形式の時刻をパース
- 今日のステータスに応じた通知内容を決定:

| ステータス | タイトル | 本文 |
| :--- | :--- | :--- |
| `kyukan` | 今日は休肝日です🍵 | よく眠れますよ。記録をつけておきましょう |
| `ok` | 今日の飲酒量を記録しましょう🍺 | 宣言した上限は守れましたか？ |
| `undecided` | 今日はどうしますか？ | まだ決めていません。今日の予定を入力しましょう |

- `DailyTrigger`（hour, minute, repeats: true）でスケジュール
- 通知タップ時の遷移先: `/(tabs)/record`

#### `cancelReminder(): Promise<void>`

- `reminder_daily` 識別子の通知をキャンセル

### 4.3. 達成通知

#### `sendAchievementNotification(): Promise<void>`

- 即時通知（trigger: null）
- タイトル: 「🎉 2連続休肝日達成！」
- 本文: 「今週の目標を達成しました。この調子で続けましょう！」
- 通知タップ時の遷移先: `/(tabs)/review`

### 4.4. 達成判定ヘルパー

#### `checkConsecutiveAchievement(records, date): boolean`

- 指定日を含む週（月曜始まり）の日付リストを取得
- `hasConsecutiveKyukan()`（`lib/store.ts`）を使って2連続休肝日の有無を判定
- `patchRecord` で status を `kyukan` に変更した直後に呼ばれる

## 5. `NotificationSetup` コンポーネント（`app/_layout.tsx`）

`AppProvider` の内側に配置されるレンダリングなし（`return null`）コンポーネント。

### 5.1. 初回権限リクエスト

- アプリ起動時に `notification_permission_requested` フラグを確認
- 未リクエストの場合のみ `requestNotificationPermission()` を呼ぶ
- Web環境ではスキップ

### 5.2. リマインダースケジュール更新

- `settings.reminderEnabled`、`settings.reminderTime`、`store` の変更を監視
- ON → `scheduleReminder()` でスケジュール更新
- OFF → `cancelReminder()` でキャンセル

### 5.3. 通知タップハンドラー

- `addNotificationResponseReceivedListener` で通知タップを検知
- 通知データの `screen` フィールドに応じて `router.push()` で画面遷移

---

## 6. 達成検知フロー（`lib/app-context.tsx`）

`patchRecord` の処理フロー:

```
patchRecord(date, { status: "kyukan" })
  ↓
updateRecord() → newStore を取得
  ↓
setStore({ ...newStore })
  ↓
patch.status === "kyukan" ?
  ↓ YES
settings.achievementNotification === true ?
  ↓ YES
checkConsecutiveAchievement(newStore.records, date) ?
  ↓ YES
sendAchievementNotification()
```

---

## 7. Web対応

すべての通知関連処理は `Platform.OS !== "web"` ガードで囲まれている。Web環境ではサイレントにスキップされ、エラーは発生しない。

## 8. 設定画面との連携

| 設定項目 | AppSettings フィールド | 通知への影響 |
| :--- | :--- | :--- |
| 夜間リマインダー ON/OFF | `reminderEnabled` | `NotificationSetup` が監視し、ON→スケジュール / OFF→キャンセル |
| リマインダー時刻 | `reminderTime` | `NotificationSetup` が監視し、時刻変更時に再スケジュール |
| 達成通知 ON/OFF | `achievementNotification` | `patchRecord` 内の判定で参照 |
