# マネタイズ戦略設計書 — 休肝日つくーる

---

## 1. 基本方針

### 現実を踏まえた収益化

「休肝日つくーる」は機能がシンプルな個人開発アプリです。
高額サブスクに見合う機能量はまだなく、**身の丈に合った価格設定**が重要です。

| 原則 | 説明 |
|:---|:---|
| 健康機能は無料 | 記録・宣言・バッジなどコア機能に課金の壁を設けない |
| 広告は非採用 | 飲酒関連広告のリスクがありアプリの趣旨と矛盾する |
| 小さく始める | まずチップ＋買い切りで検証し、機能が充実したらサブスクを検討する |
| 無理に稼がない | ユーザー基盤の成長を優先し、収益化は段階的に |

---

## 2. 推奨マネタイズモデル（段階的アプローチ）

### Phase 1（現状の機能規模）：チップ + 買い切りアンロック

サブスクは現状の機能量に対して割高感が出るため、まずは**低価格の買い切り**と**チップ（応援課金）**から始めます。

#### 2.1 チップ機能（応援課金）

設定画面に「開発者を応援する」セクションを設置。

| 金額 | ラベル |
|:---|:---|
| ¥160 | お茶を1杯 |
| ¥320 | ノンアルビールを1杯 |
| ¥800 | 晩ごはんの足しに |

- Apple/Google の手数料（15〜30%）を考慮した価格設定
- チップ後に感謝メッセージを表示（「ありがとうございます！開発の励みになります」）
- 繰り返し購入可能（Consumable IAP）
- 実装コストが低く、ユーザーの好意を自然に受け取れる

#### 2.2 買い切りプレミアムパック（¥480）

**一度買えばずっと使える**形式で、以下をまとめてアンロック。

| 機能 | 無料版 | プレミアム（買い切り¥480） |
|:---|:---|:---|
| 日々の記録・宣言 | ✅ | ✅ |
| 週間計画・バッジ5種 | ✅ | ✅ |
| 飲酒確認・代替行動 | ✅ | ✅ |
| 振り返り（週間・月間） | ✅ | ✅ |
| テーマカスタマイズ（5色） | — | ✅ |
| プレミアムバッジ（+5種） | — | ✅ |
| アプリアイコン変更 | — | ✅ |

**¥480の根拠：**
- 缶ビール1本程度の価格で心理的ハードルが低い
- 買い切りなので「元が取れるか」の不安がない
- シンプルな機能追加なので価格に見合う
- この規模のユーティリティアプリの買い切り相場（¥300〜¥600）に合致

### Phase 2（機能が充実してから）：サブスクリプション検討

以下の機能が**実際に実装・リリースされた後**に、月額プランの導入を検討します。

| 追加機能（将来） | 説明 |
|:---|:---|
| 詳細分析レポート | 飲酒パターン分析、理由トレンド、健康インパクト推定 |
| CSVエクスポート | 記録データの書き出し |
| ウィジェット | ホーム画面に今日のステータス表示 |
| 目標達成カレンダー | 月間カレンダーに色分けで達成状況を表示 |

これらが揃った段階で、**月額¥280 / 年額¥1,800**（月¥150相当）程度のサブスクが妥当です。買い切りユーザーには既存機能の永久利用を保証し、新機能のみサブスク対象とします。

## 3. Phase 1 プレミアム機能の詳細

### 3.1 テーマカスタマイズ

`theme.config.js` のカラートークンをベースに、複数のカラーテーマを提供。

| テーマ名 | 概要 |
|:---|:---|
| スタンダード | 現在のデフォルト（無料） |
| ナチュラル | グリーン系で落ち着いた配色 |
| オーシャン | ブルー系の爽やかな配色 |
| サクラ | ピンク系の温かみのある配色 |
| ミッドナイト | ダーク特化の洗練された配色 |

**実装方針：**
- `theme.config.js` を拡張し、テーマプリセットを追加
- `lib/theme-provider.tsx` でテーマ切替機能を実装
- AsyncStorageに選択テーマを保存（キー: `kyukoubi_theme_v1`）

### 3.2 プレミアムバッジ（+5種）

無料版の5種に加えて、行動パターン系のバッジを追加。

```typescript
const PREMIUM_BADGES = [
  { id: "two_month_streak",    emoji: "🏅", name: "2ヶ月継続" },
  { id: "perfect_week",        emoji: "💯", name: "完璧な1週間" },
  { id: "alternative_hero",    emoji: "🦸", name: "代替行動ヒーロー（10回選択）" },
  { id: "limit_keeper",        emoji: "🎯", name: "宣言キーパー（10回遵守）" },
  { id: "comeback",            emoji: "🔥", name: "カムバック（途切れても再開）" },
] as const;
```

### 3.3 アプリアイコン変更

Expo の `expo-alternate-app-icons` を使用し、3〜4種のアイコンから選択可能にする。

## 4. 課金基盤の技術設計

### 4.1 使用ライブラリ

```
react-native-purchases (RevenueCat)
```

**RevenueCat推奨理由：**
- iOS / Android両対応のIAP管理が統一的に可能
- レシート検証をサーバーレスで処理
- 買い切り（Non-Consumable）とチップ（Consumable）の両方に対応
- 無料枠で月$2,500収益まで対応（個人開発に十分）

### 4.2 データ設計

```typescript
// 新規追加: lib/store.ts
interface PremiumStatus {
  isPremium: boolean;           // 買い切りパック購入済みか
  purchasedAt: string | null;   // ISO 8601
}

// AsyncStorageキー: kyukoubi_premium_v1
```

### 4.3 課金ゲートの実装方針

```typescript
// lib/premium.ts（新規作成）
export function usePremium(): {
  isPremium: boolean;
  showUpgrade: () => void;   // 購入画面を表示
}
```

**UX方針：**
- 無料版でもプレミアム機能の存在を「鍵マーク」で示す
- テーマ選択画面ではプレミアムテーマのプレビューを見せてから購入を促す
- 押し付けがましい表示はしない（ペイウォールモーダルの頻度を制限）

## 5. 追加収益チャネル（将来検討）

### 5.1 法人向けB2B（中期）

- 企業の健康経営プログラム・健康保険組合との連携
- 福利厚生サービスとしての法人契約
- ユーザー基盤が一定規模に達してから検討

### 5.2 匿名統計データ（長期）

- ユーザー同意のもと、匿名・集計済みの飲酒傾向データを研究機関に提供
- 個人データの販売は絶対に行わない

## 8. 注意事項

### やってはいけないこと

| 項目 | 理由 |
|:---|:---|
| 飲酒関連の広告表示 | アプリの存在意義と矛盾する |
| コア記録機能の制限 | ユーザーの健康行動を阻害する |
| 過度な課金催促 | 個人開発アプリの信頼感を損なう |
| 個人データの外部販売 | 信頼の根本を破壊する |
| 機能不足での高額サブスク | ユーザーの不満とレビュー低下を招く |

### 法的留意点

- 特定商取引法に基づく表記の掲載
- プライバシーポリシーの整備
- 医療アドバイスではないことの免責表示
- App Store / Google Play のIAPガイドライン準拠
