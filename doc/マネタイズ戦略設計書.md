# マネタイズ戦略設計書 — 休肝日つくーる

---

## 1. 基本方針

| 原則 | 説明 |
|:---|:---|
| 機能は現状維持 | 新機能の追加・プレミアム機能の開発は行わない |
| 広告で収益化 | アプリ内にバナー広告を表示する |
| 広告非表示を課金で提供 | 広告が気になるユーザー向けに非表示オプションを販売 |
| 運用コスト最小 | 手入れを最小限にできる構成にする |

---

## 2. 広告の設計

### 2.1 広告ネットワーク

**Google AdMob** を使用。

- React Native向けに `react-native-google-mobile-ads` を導入
- iOS / Android 両対応
- バナー広告のみ（インタースティシャルやリワードは使わない）

### 2.2 広告の配置

画面下部に固定バナー広告（320×50 スタンダードバナー）を表示。

| 画面 | 広告表示 |
|:---|:---|
| ホーム | ✅ タブバーの上に固定表示 |
| 週間計画 | ✅ タブバーの上に固定表示 |
| 記録 | ✅ タブバーの上に固定表示 |
| 振り返り | ✅ タブバーの上に固定表示 |
| 設定 | ✅ タブバーの上に固定表示 |
| モーダル（宣言・確認・代替行動） | ❌ 表示しない |
| オンボーディング | ❌ 表示しない |

**モーダルに広告を出さない理由：**
- 飲酒確認画面で飲酒関連広告が出ると逆効果
- 宣言・代替行動は集中して使う画面なので邪魔になる

### 2.3 広告カテゴリの除外設定

AdMobの管理画面で以下のカテゴリをブロックする。

| ブロック対象 | 理由 |
|:---|:---|
| アルコール飲料 | アプリの趣旨と矛盾する |
| ギャンブル | 依存性のあるコンテンツを避ける |
| 出会い系 | アプリのトーンに合わない |

## 3. 広告非表示課金

### 3.1 プラン構成

| プラン | 価格 | 種別 |
|:---|:---|:---|
| 30日間広告非表示 | ¥120 | Consumable（消費型IAP） |
| 永久広告非表示 | ¥480 | Non-Consumable（非消費型IAP） |

### 3.2 購入導線

設定画面に「広告を非表示にする」セクションを追加。

```
┌─────────────────────────────────┐
│  広告を非表示にする              │
│                                 │
│  [30日間 ¥120]  [永久 ¥480]     │
│                                 │
│  ※ 購入の復元はこちら            │
└─────────────────────────────────┘
```

加えて、バナー広告の端に小さな「×広告を消す」リンクを配置し、設定画面の該当セクションへ遷移させる。

### 3.3 データ設計

```typescript
// lib/store.ts に追加
interface AdFreeStatus {
  type: "none" | "temporary" | "permanent";
  expiresAt: string | null;  // "temporary" の場合のみ ISO 8601
}

// AsyncStorageキー: kyukoubi_adfree_v1
// デフォルト値: { type: "none", expiresAt: null }
```

### 3.4 技術設計

```typescript
// lib/ad-free.ts（新規作成）
export function useAdFree(): {
  isAdFree: boolean;        // 広告非表示中か
  status: AdFreeStatus;
  purchase30Day: () => void;
  purchasePermanent: () => void;
  restorePurchase: () => void;
}
```

**判定ロジック：**
```typescript
function isAdFree(status: AdFreeStatus): boolean {
  if (status.type === "permanent") return true;
  if (status.type === "temporary" && status.expiresAt) {
    return new Date(status.expiresAt) > new Date();
  }
  return false;
}
```

### 3.5 課金ライブラリ

```
react-native-purchases (RevenueCat)
```

- 買い切り（Non-Consumable）と消費型（Consumable）の両方に対応
- レシート検証をサーバーレスで処理
- 「購入の復元」機能が標準で提供される
- 無料枠で月$2,500収益まで対応

## 4. 実装方針

### 4.1 広告コンポーネント

```typescript
// components/ad-banner.tsx（新規作成）
// タブ画面で共通利用するバナー広告コンポーネント
// useAdFree() で非表示判定し、広告非表示中は何もレンダリングしない
```

**配置方法：**
- `app/(tabs)/_layout.tsx` のタブバーの上にバナーを固定表示
- 各タブ画面のコンテンツ下部に `paddingBottom` を追加して広告と重ならないようにする

### 4.2 変更が必要なファイル

| ファイル | 変更内容 |
|:---|:---|
| `lib/store.ts` | `AdFreeStatus` 型の追加 |
| `lib/ad-free.ts` | 新規作成：`useAdFree` フック |
| `components/ad-banner.tsx` | 新規作成：バナー広告コンポーネント |
| `app/(tabs)/_layout.tsx` | バナー広告の配置 |
| `app/(tabs)/settings.tsx` | 「広告を非表示にする」セクション追加 |
| `app/_layout.tsx` | RevenueCat の初期化 |
| `app.config.ts` | AdMob App ID の設定 |

## 5. 注意事項

| 項目 | 対応 |
|:---|:---|
| 飲酒関連広告のブロック | AdMob管理画面でアルコールカテゴリを除外 |
| GDPR / ATT対応 | `expo-tracking-transparency` でiOS ATTダイアログを表示 |
| 子供向け設定 | COPPA対象外の設定（飲酒管理アプリのため成人向け） |
| 購入の復元 | App Storeガイドラインで必須。設定画面に「購入の復元」を配置 |
| 広告の読み込み失敗 | 失敗時は広告枠を非表示にし、レイアウトが崩れないようにする |
