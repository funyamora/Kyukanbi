# マネタイズ戦略設計書 — 休肝日つくーる

---

## 1. 基本方針

### コアバリューを無料で維持する

「休肝日つくーる」は**健康習慣の形成**を目的としたアプリです。
マネタイズにおいて最も重要なのは、**ユーザーの健康行動を阻害しないこと**です。

以下の原則に従います。

| 原則 | 説明 |
|:---|:---|
| 健康機能は無料 | 休肝日の記録・宣言・バッジなどコア機能は課金の壁を設けない |
| 広告は非採用 | 飲酒関連広告が表示されるリスクがあり、アプリの趣旨と矛盾する |
| 段階的導入 | まず無料でユーザー基盤を確立し、プレミアム機能を段階的に追加する |

## 3. プレミアム機能の詳細設計

### 3.1 詳細分析レポート（振り返り画面の強化）

無料版の振り返り画面を拡張し、より深い洞察を提供します。

**追加される分析項目：**

| 分析項目 | 内容 |
|:---|:---|
| 飲酒パターン分析 | 曜日×時間帯のヒートマップ、季節・イベント傾向 |
| 理由トレンド | 飲酒理由の月次推移グラフ |
| 目標達成カレンダー | 月間カレンダーに色分けで達成状況を表示 |
| 健康インパクト推定 | 休肝日による推定カロリー削減・睡眠改善の可視化 |
| 連続記録ストリーク | アプリ利用日数の連続記録 |

**実装方針：**
- 既存の `computeWeekdayAverages` と `computeMonthlySummary` を拡張
- すべてローカル計算（バックエンド不要）
- `lib/store.ts` に新しい分析関数を追加

### 3.3 CSVエクスポート

振り返り画面から飲酒記録をCSVファイルとして出力する機能。

**出力項目：**
```
日付,ステータス,宣言上限,実際の杯数,満足度,理由,代替行動,メモ
2026-03-01,kyukan,,,,,,"体調良い"
2026-03-02,ok,2,1,great,ストレス,,""
```

**実装方針：**
- `expo-file-system` + `expo-sharing` を使用
- 期間選択UI（1週間 / 1ヶ月 / 全期間）
- ヘルスケアアプリとの連携データとしても活用可能

### 3.4 テーマカスタマイズ

`theme.config.js` のカラートークンをベースに、複数のカラーテーマを提供。

| テーマ名 | 概要 |
|:---|:---|
| スタンダード | 現在のデフォルト（無料） |
| ナチュラル | グリーン系で落ち着いた配色 |
| オーシャン | ブルー系の爽やかな配色 |
| サクラ | ピンク系の温かみのある配色 |
| ミッドナイト | ダーク特化の洗練された配色 |

**実装方針：**
- `theme.config.js` を拡張し、テーマプリセットを追加
- `lib/theme-provider.tsx` でテーマ切替機能を実装
- AsyncStorageに選択テーマを保存（キー: `kyukoubi_theme_v1`）

### 3.5 ウィジェット（iOS / Android）

ホーム画面に今日のステータスと週間進捗を表示するウィジェット。

**表示内容：**
- 今日のステータス（休肝日 / 飲酒OK / 未定）
- 今週の休肝日カウント / 目標
- 連続休肝日数

**実装方針：**
- `react-native-widget-extension`（iOS）または `expo-widgets`（検討中）

## 4. 課金基盤の技術設計

### 4.1 使用ライブラリ

```
expo-in-app-purchases  または  react-native-purchases (RevenueCat)
```

**RevenueCat推奨理由：**
- iOS / Android両対応のサブスク管理が統一的に可能
- レシート検証をサーバーレスで処理
- ダッシュボードでMRR・チャーン等の指標を確認可能
- 無料枠で月$2,500収益まで対応（初期フェーズに十分）

### 4.2 データ設計

```typescript
// 新規追加: lib/store.ts
interface PremiumStatus {
  isPremium: boolean;
  expiresAt: string | null;  // ISO 8601
  plan: "monthly" | "yearly" | null;
}

// AsyncStorageキー: kyukoubi_premium_v1
```

### 4.3 課金ゲートの実装方針

```typescript
// lib/premium.ts（新規作成）
export function usePremium(): {
  isPremium: boolean;
  showPaywall: () => void;
}

// 使用例：振り返り画面で過去月を表示しようとした時
const { isPremium, showPaywall } = usePremium();

if (!isPremium) {
  showPaywall();
  return;
}
```

**UX方針：**
- ハードペイウォールは使わない（機能を完全にブロックしない）
- 無料版でもプレミアム機能の「プレビュー」を見せ、価値を実感させてから課金を促す
- 例：月間レポートは「当月のみ無料、過去月はプレミアム」

## 8. 注意事項

### やってはいけないこと

| 項目 | 理由 |
|:---|:---|
| 飲酒関連の広告表示 | アプリの存在意義と矛盾する |
| コア記録機能の制限 | ユーザーの健康行動を阻害する |
| 過度なペイウォール表示 | ユーザー離脱の原因になる |
| 個人データの外部販売 | 信頼の根本を破壊する |
| ゲーミフィケーションの課金依存化 | バッジのために飲酒する本末転倒を防ぐ |

### 法的留意点

- 特定商取引法に基づく表記の掲載
- サブスク自動更新の明確な説明（App Store / Google Play ガイドライン準拠）
- プライバシーポリシーの整備（匿名データ活用時は特に重要）
- 医療アドバイスではないことの免責表示
