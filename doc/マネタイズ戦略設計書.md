# マネタイズ戦略設計書 — 休肝日つくーる

---

## 1. 基本方針

| 原則 | 説明 |
|:---|:---|
| 機能は現状維持 | 新機能の追加・プレミアム機能の開発は行わない |
| 広告で収益化 | アプリ内に複数フォーマットの広告を表示する |
| 広告非表示を課金で提供 | 広告が気になるユーザー向けに非表示オプションを販売 |
| チップで応援課金 | 気に入ったユーザーが気軽に支援できる導線を用意する |
| 運用コスト最小 | 手入れを最小限にできる構成にする |

---

## 2. 広告の設計

### 2.1 広告ネットワーク・ライブラリ

**Google AdMob** を使用。

- React Native向けに `react-native-google-mobile-ads`（Invertase製）を導入
- iOS / Android 両対応
- バナー・インタースティシャル・App Open Ads・ネイティブ広告に対応

### 2.2 広告フォーマットと配置

#### バナー広告（常時表示）

画面下部に固定バナー広告（320×50 スタンダードバナー）を表示。

| 画面 | 広告表示 |
|:---|:---|
| ホーム | ✅ タブバーの上に固定表示 |
| 週間計画 | ✅ タブバーの上に固定表示 |
| 記録 | ✅ タブバーの上に固定表示 |
| 振り返り | ✅ タブバーの上に固定表示 |
| 設定 | ✅ タブバーの上に固定表示 |
| モーダル（宣言・確認・代替行動） | ❌ 表示しない |
| オンボーディング | ❌ 表示しない |

#### App Open Ads（起動時広告）

アプリ起動時（コールドスタート・バックグラウンド復帰）に全画面広告を表示。

| 項目 | 設定 |
|:---|:---|
| 表示タイミング | スプラッシュ画面の直後 |
| バックグラウンド復帰時 | 30分以上バックグラウンドにいた場合のみ表示 |
| 頻度制限 | 1日最大3回 |
| 初回起動 | 表示しない（オンボーディングを優先） |

**実装方針：**
- `app/_layout.tsx` で `AppState` の変化を監視
- バックグラウンド→フォアグラウンド復帰時に経過時間を判定して表示

#### インタースティシャル広告（全画面・画面遷移時）

自然な操作の区切りで全画面広告を表示。

| 表示タイミング | 説明 |
|:---|:---|
| 飲酒宣言の完了後 | 「宣言して飲む」を押した後、ホームに戻る前 |
| 記録の保存後 | 飲酒量・満足度を記録した後 |

**表示しないタイミング：**

| タイミング | 理由 |
|:---|:---|
| 飲酒確認画面の前後 | 「本当に飲みますか？」の判断を邪魔しない |
| 代替行動の選択後 | 休肝日を選んだ良い行動を広告で台無しにしない |
| 休肝日宣言時 | ポジティブな行動の直後に広告はストレスになる |

**頻度制限：**
- 1日最大2回まで
- 前回表示から最低3分間はインターバルを空ける

#### ネイティブ広告（コンテンツ内埋め込み）

アプリのデザインに溶け込む広告をコンテンツ内に配置。

| 配置場所 | 説明 |
|:---|:---|
| 振り返り画面（週間タブ） | KPIカードとグラフの間に1枠 |
| 設定画面 | セクション間に1枠 |

**実装方針：**
- `react-native-google-mobile-ads` の `NativeAd` コンポーネントを使用
- アプリのカード UI に合わせたカスタムレイアウトを作成
- NativeWind のスタイリングに統一し、周囲のコンテンツと自然に馴染ませる
- 「広告」ラベルを必ず表示（AdMobポリシー要件）

**優先度：低（後回し可）**
- バナー・インタースティシャル・App Open Adsより実装コストが高い
- 広告レイアウト（タイトル・説明文・画像・CTAボタン）を自前でデザインする必要がある
- 他の広告フォーマットを先に実装し、余力があれば追加する

### 2.3 広告カテゴリの除外設定

AdMobの管理画面で以下のカテゴリをブロックする。

| ブロック対象 | 理由 |
|:---|:---|
| アルコール飲料 | アプリの趣旨と矛盾する |
| ギャンブル | 依存性のあるコンテンツを避ける |
| 出会い系 | アプリのトーンに合わない |

## 3. 広告非表示課金

### 3.1 プラン構成

| プラン | 価格 | 種別 |
|:---|:---|:---|
| 30日間広告非表示 | ¥120 | Consumable（消費型IAP） |
| 永久広告非表示 | ¥480 | Non-Consumable（非消費型IAP） |

購入するとすべてのフォーマット（バナー・インタースティシャル・App Open Ads・ネイティブ広告）が非表示になる。

### 3.2 購入導線

設定画面に「広告を非表示にする」セクションを追加。

```
┌─────────────────────────────────┐
│  広告を非表示にする              │
│                                 │
│  [30日間 ¥120]  [永久 ¥480]     │
│                                 │
│  ※ 購入の復元はこちら            │
└─────────────────────────────────┘
```

加えて、バナー広告の端に小さな「×広告を消す」リンクを配置し、設定画面の該当セクションへ遷移させる。

### 3.3 データ設計

```typescript
// lib/store.ts に追加
interface AdFreeStatus {
  type: "none" | "temporary" | "permanent";
  expiresAt: string | null;  // "temporary" の場合のみ ISO 8601
}

// AsyncStorageキー: kyukoubi_adfree_v1
// デフォルト値: { type: "none", expiresAt: null }
```

### 3.4 技術設計

```typescript
// lib/ad-free.ts（新規作成）
export function useAdFree(): {
  isAdFree: boolean;        // 広告非表示中か
  status: AdFreeStatus;
  purchase30Day: () => void;
  purchasePermanent: () => void;
  restorePurchase: () => void;
}
```

**判定ロジック：**
```typescript
function isAdFree(status: AdFreeStatus): boolean {
  if (status.type === "permanent") return true;
  if (status.type === "temporary" && status.expiresAt) {
    return new Date(status.expiresAt) > new Date();
  }
  return false;
}
```

## 4. チップ機能（応援課金）

### 4.1 概要

アプリを気に入ったユーザーが開発者を気軽に支援できるチップ機能。
広告非表示とは独立した、純粋な応援課金として提供する。

### 4.2 プラン構成

| 金額 | ラベル | 種別 |
|:---|:---|:---|
| ¥160 | お茶を1杯 | Consumable（消費型IAP） |
| ¥320 | ノンアルビールを1杯 | Consumable（消費型IAP） |
| ¥800 | 晩ごはんの足しに | Consumable（消費型IAP） |

- すべて消費型IAP（繰り返し購入可能）
- Apple/Googleの手数料（15〜30%）を考慮した価格設定

### 4.3 UI設計

設定画面の下部に「開発者を応援する」セクションを配置。

```
┌─────────────────────────────────────┐
│  開発者を応援する                    │
│                                     │
│  このアプリが役に立っていたら、      │
│  開発の励みに応援していただけると    │
│  嬉しいです。                       │
│                                     │
│  [お茶を1杯 ¥160]                   │
│  [ノンアルビールを1杯 ¥320]         │
│  [晩ごはんの足しに ¥800]            │
└─────────────────────────────────────┘
```

### 4.4 購入後の体験

- 購入完了時に感謝メッセージをアラートで表示
  - 「ありがとうございます！開発の励みになります」
- ハプティクス: `Haptics.notificationAsync(Success)`
- 購入回数などの記録・表示は行わない（シンプルに保つ）

### 4.5 実装方針

- 広告非表示と同じく RevenueCat で管理
- Consumable IAP として設定
- 設定画面にボタンを並べるだけなので実装コストは低い

## 5. 課金基盤の技術設計

### 5.1 使用ライブラリ

```
react-native-purchases (RevenueCat)
```

**RevenueCat推奨理由：**
- iOS / Android両対応のIAP管理が統一的に可能
- レシート検証をサーバーレスで処理
- 買い切り（Non-Consumable）と消費型（Consumable）の両方に対応
- 「購入の復元」機能が標準で提供される
- 無料枠で月$2,500収益まで対応

### 5.2 IAP商品一覧

| `kyukanbi_adfree_permanent` | Non-Consumable | ¥480 | 永久広告非表示 |
| `kyukanbi_tip_small` | Consumable | ¥160 | チップ：お茶を1杯 |
| `kyukanbi_tip_medium` | Consumable | ¥320 | チップ：ノンアルビールを1杯 |

```typescript
// components/ad-banner.tsx（新規作成）
// タブ画面で共通利用するバナー広告コンポーネント
```

## 7. 注意事項

| 項目 | 対応 |
|:---|:---|
| 飲酒関連広告のブロック | AdMob管理画面でアルコールカテゴリを除外 |
| GDPR / ATT対応 | `expo-tracking-transparency` でiOS ATTダイアログを表示 |
| 子供向け設定 | COPPA対象外の設定（飲酒管理アプリのため成人向け） |
| 購入の復元 | App Storeガイドラインで必須。設定画面に「購入の復元」を配置 |
| 広告の読み込み失敗 | 失敗時は広告枠を非表示にし、レイアウトが崩れないようにする |
| インタースティシャルの頻度 | 1日最大2回・最低3分間隔で、ユーザー体験を損なわない |
| App Open Adsの初回 | オンボーディング中は表示しない |
| Expo Go非対応 | `react-native-google-mobile-ads` はExpo Go非対応。EAS Build または `npx expo prebuild` を使用 |
