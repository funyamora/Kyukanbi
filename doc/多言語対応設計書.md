# 多言語対応（i18n）設計書

## 1. 概要

「休肝日つくーる」を多言語対応し、日本語以外のユーザーにも利用可能にする。
初期対応言語は **日本語（ja）** と **英語（en）** の2言語とする。

---

## 2. 技術選定

### 採用ライブラリ

| パッケージ | 用途 |
| :--- | :--- |
| `i18next` | 翻訳エンジン（キーベースの文字列管理） |
| `react-i18next` | Reactコンポーネントとの統合（`useTranslation` フック） |
| `expo-localization` | デバイスの言語設定を取得 |

### 選定理由

- `i18next` はReact Nativeで広く使われ、エコシステムが成熟している
- 名前空間（namespace）で翻訳ファイルを画面単位に分割でき、管理しやすい
- 複数形・補間（interpolation）・ネスト対応など機能が豊富
- `expo-localization` はExpo公式でデバイス言語の検出が簡単

## 5. i18next初期化（lib/i18n.ts）

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import { getLocales } from "expo-localization";

// 翻訳リソースのインポート（各言語・各名前空間）
import jaCommon from "./locales/ja/common.json";
import enCommon from "./locales/en/common.json";
// ... 他の名前空間も同様

const deviceLang = getLocales()[0]?.languageCode ?? "ja";

i18n.use(initReactI18next).init({
  resources: {
    ja: { common: jaCommon, /* ... */ },
    en: { common: enCommon, /* ... */ },
  },
  lng: deviceLang,             // デバイス言語を初期値に
  fallbackLng: "ja",           // フォールバックは日本語
  defaultNS: "common",         // デフォルト名前空間
  ns: ["common", "tabs", "home", "weekly", "record", "review",
       "settings", "onboarding", "declaration", "alternative",
       "confirm", "badges", "notifications"],
  interpolation: {
    escapeValue: false,         // React Nativeではエスケープ不要
  },
});

export default i18n;
```

## 6. 使い方

### 6.1 画面コンポーネントでの使用

```typescript
import { useTranslation } from "react-i18next";

export default function HomeScreen() {
  const { t } = useTranslation("home");

  return (
    <ScreenContainer>
      <Text className="text-foreground text-2xl font-bold">
        {t("title")}
      </Text>
      <Text>{t("common:status.kyukan")}</Text>
    </ScreenContainer>
  );
}
```

### 6.2 非コンポーネント（通知・store関数）での使用

```typescript
import i18n from "@/lib/i18n";

// コンポーネント外ではi18nインスタンスを直接使う
const title = i18n.t("notifications:reminder.kyukan.title");
```

### 6.3 日付フォーマット

`formatDateJP` を `formatDate` にリネームし、i18nの補間を使う:

```typescript
function formatDate(dateStr: string): string {
  const d = new Date(dateStr);
  const dayKeys = ["sun", "mon", "tue", "wed", "thu", "fri", "sat"];
  const dayOfWeek = i18n.t(`common:days.${dayKeys[d.getDay()]}`);

  return i18n.t("common:date.format", {
    month: d.getMonth() + 1,
    day: d.getDate(),
    dayOfWeek,
    monthName: d.toLocaleDateString(i18n.language, { month: "long" }),
    year: d.getFullYear(),
  });
}
```

## 8. 言語切替機能

### 8.1 設定画面に言語選択を追加

設定画面（`app/(tabs)/settings.tsx`）に言語選択セクションを追加する。

```
┌─ 言語 / Language ──────────────────┐
│  🇯🇵 日本語                    ✓   │
│  🇬🇧 English                       │
└────────────────────────────────────┘
```

### 8.2 言語設定の永続化

`AppSettings` に `language` フィールドを追加:

```typescript
interface AppSettings {
  // ... 既存フィールド
  language: "ja" | "en" | "auto"; // デフォルト: "auto"（デバイス設定に従う）
}
```

AsyncStorageに保存し、アプリ起動時に復元する。

## 9. 実装順序

以下の順序で段階的に実装する。各ステップ完了時に `pnpm check` と `pnpm test` を実行して確認する。

### Phase 1: 基盤構築

1. `i18next`、`react-i18next`、`expo-localization` をインストール
2. `lib/i18n.ts` を作成（初期化コード）
3. `lib/locales/ja/common.json` を作成（共通翻訳キー）
4. `lib/locales/en/common.json` を作成（英語翻訳）
5. `app/_layout.tsx` にi18nインポートを追加

### Phase 2: 画面ごとの翻訳リソース作成

6. 各画面の日本語翻訳JSONを作成（既存のハードコード文字列を抽出）
7. 各画面の英語翻訳JSONを作成

### Phase 3: 画面コンポーネントの置換

8. `app/(tabs)/_layout.tsx` のタブ名を置換
9. 各画面のハードコード文字列を `t()` に順次置換
10. `lib/store.ts` のバッジ名・日付フォーマット・コメントを翻訳化
11. `lib/notifications.ts` の通知メッセージを翻訳化
12. `components/BadgeList.tsx`、`components/WeekdayChart.tsx` を翻訳化

### Phase 4: 言語切替機能

13. `AppSettings` に `language` フィールドを追加
14. 設定画面に言語切替UIを実装
15. アプリ起動時の言語復元ロジックを実装

### Phase 5: テスト・検証

16. 既存テストの修正（翻訳キーのモック対応）
17. 言語切替の動作確認テストを追加
18. 全画面の英語表示を目視確認

## 11. 注意事項

### 翻訳キーの命名規則

- スネークケースを使用: `weekly_goal`, `drink_count`
- 階層はドットで区切る: `status.kyukan`, `button.save`
- 画面固有のキーは画面名の名前空間に配置する

### 翻訳不要な要素

- 絵文字（全言語共通）
- アプリ内部のID・キー（`kyukoubi_store_v1` 等）
- 型定義のフィールド名
- カラーコード・数値

### バッジ定義の変更

`BADGE_DEFINITIONS` の `name` フィールドは翻訳キーに置き換える:

```typescript
// Before
{ id: "first_kyukan", emoji: "🥉", name: "初めての休肝日" }

// After
{ id: "first_kyukan", emoji: "🥉", nameKey: "first_kyukan" }
// 表示時: t("badges:first_kyukan") → "初めての休肝日" or "First Alcohol-Free Day"
```

### 将来の言語追加

新しい言語を追加する場合は:

1. `lib/locales/<lang>/` ディレクトリを作成
2. 全名前空間のJSONファイルを翻訳して配置
3. `lib/i18n.ts` の `resources` に追加
4. `AppSettings` の `language` 型に追加
5. 設定画面の言語リストに追加

## 12. アプリ名の多言語対応

`app.config.ts` でロケール別のアプリ名を設定する:

```typescript
// iOS: InfoPlist.strings で "休肝日つくーる" / "Kyukanbi Maker" を切替
// Android: values/strings.xml, values-en/strings.xml で切替
```

ただし、アプリ名の多言語化はストア申請にも関わるため、Phase 4以降の対応で良い。
