# 記録画面の杯数選択肢を2段階UIに変更 - 2026年03月08日 15:55

## 作業内容
記録画面の杯数選択で「4杯以上」を選んだときにサブ選択肢（4〜7杯、8杯以上）をアニメーション付きで展開表示するよう変更。これにより最大4杯までしか記録できなかった制限を解消。

## 変更ファイル
- app/(tabs)/record.tsx

## 変更詳細

### DRINK_DETAIL_OPTIONS定数の追加
「4杯以上」選択時に表示されるサブ選択肢を定義。4杯、5杯、6杯、7杯、8杯以上の5つの選択肢。

### 2段階選択ロジックの実装
- 1〜3杯: 従来通り即座に`actualDrinks`にセット、サブ選択肢は閉じる
- 「4杯以上」ボタン: サブ選択肢をアニメーション展開（デフォルトで4を選択）
- サブ選択肢: 選んだ値を`actualDrinks`にセット
- `actualDrinks >= 4`の場合、「4杯以上」ボタンが選択状態＋サブ選択肢の該当ボタンもハイライト

### アニメーション
`react-native-reanimated`の`withTiming`を使用。展開150ms、閉じる120ms。高さとopacityをアニメーション。

### 変更箇所（行番号）
```typescript
// 追加: サブ選択肢の定数
const DRINK_DETAIL_OPTIONS = [
  { value: 4, label: "4杯" },
  { value: 5, label: "5杯" },
  { value: 6, label: "6杯" },
  { value: 7, label: "7杯" },
  { value: 8, label: "8杯以上" },
];

// 変更: 「4杯以上」の選択状態判定
// 変更前: const isSelected = actualDrinks === opt.value;
// 変更後:
const isSelected =
  opt.value === 4
    ? actualDrinks !== null && actualDrinks >= 4
    : actualDrinks === opt.value;
```

## 影響範囲
- 記録画面の杯数選択UI
- `actualDrinks`の値範囲が1〜4から0〜8に拡大（データモデルの`actualDrinks: number | null`は変更不要）
