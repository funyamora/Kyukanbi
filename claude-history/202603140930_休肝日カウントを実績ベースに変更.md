# 休肝日カウントを計画ベースから実績ベースに変更 - 2026年03月14日 09:30

## 作業内容
休肝日の判定を `status === "kyukan"`（計画ベース）から `actualDrinks === 0`（実績ベース）に変更。計画しただけで「達成済み」になる問題を修正した。

## 変更ファイル
- lib/store.ts
- app/(tabs)/review.tsx
- tests/analytics.test.ts
- tests/store.test.ts
- tests/notifications.test.ts
- CLAUDE.md

## 変更詳細

### isConfirmedKyukan ヘルパー関数の追加（lib/store.ts）
実績ベースで休肝日確定を判定するヘルパー関数を新規追加。`actualDrinks === 0` で判定する。

### 休肝日判定の全箇所を実績ベースに変更（lib/store.ts）
- `hasConsecutiveKyukan`: 2連続休肝日判定
- `canAchieveConsecutiveIfDrink`: 飲酒シミュレーション（`actualDrinks: 1` を設定）
- `weekGoalMet`: 週間目標達成判定
- `checkNewBadges`: バッジ解除判定（first_kyukan, first_consecutive）
- `computeMonthlySummary`: 月次サマリーの休肝日カウント

### 振り返り画面の休肝日カウント変更（app/(tabs)/review.tsx）
- `computeStats` 内の `kyukanDays` と `isKyukan` を `isConfirmedKyukan` で判定

### テストデータの修正
全テストファイルのテストデータに `actualDrinks: 0`（休肝日）や `actualDrinks: 2`（飲酒日）を追加し、実績ベースの判定に対応。

### 変更箇所（行番号）
```typescript
// 変更前（lib/store.ts 各所）
records[key]?.status === "kyukan"

// 変更後
isConfirmedKyukan(records[key])
```

## 影響範囲
- ホーム画面の2連続休肝日達成表示
- バッジシステム全体（first_kyukan, first_consecutive, first_weekly_goal, three_weeks_streak, one_month_streak）
- 振り返り画面の休肝日カウント・グラフ表示
- 月次サマリーの休肝日数・達成率
