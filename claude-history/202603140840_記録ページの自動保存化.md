# 記録ページの自動保存化 - 2026年03月14日 08:40

## 作業内容
記録画面（`app/(tabs)/record.tsx`）の「記録を保存する」ボタンを廃止し、各項目の変更時に自動保存するように変更した。

## 変更ファイル
- `app/(tabs)/record.tsx`
- `doc/画面設計書.md`

## 変更詳細

### 杯数・満足度の即時保存
杯数（1〜3杯、4杯以上の詳細選択、0杯）および満足度の各`onPress`ハンドラで、ローカルstate更新後に`patchRecord`を呼び出して即座にAsyncStorageへ保存するようにした。

### メモのonBlur保存
TextInputに`onBlur`ハンドラを追加し、フォーカスが外れた時点で`patchRecord(selectedDate, { memo })`を実行。入力中は毎文字ごとに保存しない（パフォーマンス配慮）。

### 保存ボタン・関連コードの削除
- `handleSave`コールバック削除
- `saved` state削除
- 画面下部の`bottomBar`（保存ボタン含むView）削除
- StyleSheetから`bottomBar`, `saveBtn`, `saveBtnText`を削除
- ScrollViewの`paddingBottom`を`insets.bottom + 100` → `insets.bottom + 20`に縮小

### ドキュメント更新
`doc/画面設計書.md`の記録画面セクションに自動保存の挙動を追記。

### 変更箇所（行番号）
```typescript
// 削除: saved state
- const [saved, setSaved] = useState(false);

// 削除: handleSave コールバック
- const handleSave = useCallback(async () => { ... });

// 追加: 杯数選択時の即時保存（例: 1〜3杯）
onPress={() => {
  setActualDrinks(opt.value);
  closeDetailOptions();
+ patchRecord(selectedDate, { actualDrinks: opt.value });
}}

// 追加: 満足度選択時の即時保存
onPress={() => {
  setSatisfaction(opt.value);
+ patchRecord(selectedDate, { satisfaction: opt.value });
}}

// 追加: メモのonBlur保存
+ onBlur={() => patchRecord(selectedDate, { memo })}

// 削除: bottomBar（保存ボタン）全体
```

## 影響範囲
- 記録画面のUX（保存ボタン不要に）
- バッジ判定・達成通知のタイミング（項目変更ごとに発火）
