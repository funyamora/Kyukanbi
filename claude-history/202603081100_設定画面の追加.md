# 設定画面の追加 - 2026年03月08日 11:00

## 作業内容
5番目のタブとして設定画面（Settings）を追加。目標設定・通知設定（UIのみ）・データ管理の3セクションを実装した。

## 変更ファイル
- `lib/store.ts`（変更）
- `lib/app-context.tsx`（変更）
- `app/(tabs)/settings.tsx`（新規）
- `app/(tabs)/_layout.tsx`（変更）
- `app/(tabs)/index.tsx`（変更）
- `tests/store.test.ts`（変更）
- `doc/設定画面設計書.md`（新規）

## 変更詳細

### AppSettings 型と永続化関数の追加（store.ts）
- `AppSettings` インターフェースを定義（weeklyGoalDays, requireConsecutive, reminderEnabled, reminderTime, achievementNotification）
- `defaultSettings()` でデフォルト値を生成
- `loadSettings()` / `saveSettings()` で AsyncStorage 永続化
- `clearAllData()` で records と settings の両方を削除
- AsyncStorage キー: `kyukoubi_settings_v1`

### Context 拡張（app-context.tsx）
- `settings` state を追加
- `patchSettings(patch)` で設定の部分更新（マージ → 保存 → state更新）
- `resetAllData()` で全データリセット（store + settings を初期化）

### 設定画面の実装（settings.tsx）
- 目標設定セクション: 週の目標休肝日数（ステッパー 1〜5）、2連続休肝日トグル
- 通知設定セクション: 夜間リマインダー、リマインダー時刻表示、達成通知（Phase 2注記付き）
- データ管理セクション: 全データリセットボタン（確認ダイアログ付き）
- Web対応: `Alert.alert()` が Web で動作しないため `window.confirm()` にフォールバック
- Pressable は style prop を使用（className の onPress バグ回避）

### タブ追加（_layout.tsx）
- 5番目のタブ `settings` を追加（アイコン: gearshape.fill、タイトル: 設定）

### ホーム画面の設定ボタン接続（index.tsx）
- ヘッダーの ⚙️ ボタンの onPress を `router.push("/(tabs)/settings")` に変更

### テスト追加（store.test.ts）
- `defaultSettings` のデフォルト値テスト
- `defaultSettings` の参照独立性テスト

## 影響範囲
- タブバー: 4タブ → 5タブに変更
- ホーム画面: 設定ボタンが機能するようになった
- Context: settings / patchSettings / resetAllData が全画面で利用可能に
