## 変更詳細

### バグ1: ステータスバッジの色ずれ（Web）
連続タップでステータスを切り替えると、ラベルは正しいのに色だけ古いまま残る問題。
React Native Web のインラインスタイルキャッシュが原因。

```tsx
// 変更前（インラインオブジェクト → キャッシュ問題）
<View style={[styles.statusBadge, { backgroundColor: conf.bg, borderColor: conf.border }]}>
<Text style={[styles.statusBadgeText, { color: conf.text }]}>

// 変更後（事前定義StyleSheet → CSSクラスベースで安全）
const STATUS_BADGE_STYLES = StyleSheet.create({
  kyukan:    { backgroundColor: "#E8F5E9", borderColor: "#4CAF50" },
  ok:        { backgroundColor: "#FFF3E0", borderColor: "#FF6B35" },
  undecided: { backgroundColor: "#F2F2F7", borderColor: "#E5E5EA" },
});
<View style={[styles.statusBadge, STATUS_BADGE_STYLES[rec.status]]}>
```

### バグ2: スワイプ後にタップが発火してステータスが上書きされる（Web）
右スワイプで休肝日に設定→直後にPressableのonPressが発火→サイクル切替で飲酒OKに上書きされる問題。

```tsx
// swipedRef フラグでスワイプ確定を記録し、直後のonPressを無視
const swipedRef = useRef(false);
// Pan.onEnd でフラグを立てる
runOnJS(markSwiped)();
// handlePress でフラグチェック
if (swipedRef.current) { swipedRef.current = false; return; }
```

### バグ3: スワイプ背景ラベルの見切れ
SWIPE_CLAMPが80pxでラベル幅に対して不十分だった。

```tsx
// 変更前
const SWIPE_THRESHOLD = 60;
const SWIPE_CLAMP = 80;

// 変更後
const SWIPE_THRESHOLD = 80;
const SWIPE_CLAMP = 120;
```

### その他の修正
- `Gesture.Race(tap, pan)` → Pan のみに変更、タップは Pressable に分離
- `Animated.View` とコンテンツ View を分離（スタイル更新の信頼性向上）
- `handleToggle` に楽観的ロック（`togglingRef`）追加
- `FlatList` に `extraData={store.records}` 追加
- `lib/store.ts` の `clearAllData` に `onboarding_completed` の削除を追加
