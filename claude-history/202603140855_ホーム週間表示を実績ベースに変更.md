# ホーム「今週の休肝日」表示を実績ベースに変更 - 2026年03月14日 08:55

## 作業内容
ホーム画面の「今週の休肝日」セクションで、過去の日付の表示ロジックを計画ベースから実績ベースに変更。記録（actualDrinks）の有無に基づいて表示ステータスを決定するようにした。

## 変更ファイル
- app/(tabs)/index.tsx
- doc/画面設計書.md

## 変更詳細

### 表示ステータスの決定ロジック追加
従来は全日付で `rec.status`（計画ステータス）をそのまま表示していたが、過去の日付は `actualDrinks` の値から表示ステータスを導出するように変更。

| 日付 | actualDrinks | 表示 |
|------|-------------|------|
| 昨日以前 | null | undecided（？） |
| 昨日以前 | 0 | kyukan（🍵） |
| 昨日以前 | 1以上 | ok（🥃） |
| 今日 | null | 計画の status |
| 今日 | 0 | kyukan（🍵） |
| 今日 | 1以上 | ok（🥃） |
| 明日以降 | — | 計画の status |

### 変更箇所（行番号）
```typescript
// 変更前（app/(tabs)/index.tsx:201）
const conf = STATUS_CONFIG[rec.status];

// 変更後（app/(tabs)/index.tsx:201-209）
const displayStatus: DayStatus =
  date < today
    ? rec.actualDrinks !== null
      ? rec.actualDrinks === 0 ? "kyukan" : "ok"
      : "undecided"
    : date === today && rec.actualDrinks !== null
      ? rec.actualDrinks === 0 ? "kyukan" : "ok"
      : rec.status;
const conf = STATUS_CONFIG[displayStatus];
```

### 画面設計書の更新
週の進捗セクションの説明に、日付ごとの表示ロジックの詳細を追記。

## 影響範囲
- ホーム画面の「今週の休肝日」セクションの表示のみ
- バッジ判定・週間計画画面・記録画面には影響なし

## テスト項目
1. `pnpm check` で型エラーなし ✅
2. `pnpm test` でテスト全件パス ✅
3. 昨日以前で未記録の日 → 「？」表示
4. 昨日以前で杯数記録済みの日 → 実績に応じた emoji 表示
5. 今日は記録があれば実績ベース、なければ計画ベース
6. 明日以降は計画のステータス表示
