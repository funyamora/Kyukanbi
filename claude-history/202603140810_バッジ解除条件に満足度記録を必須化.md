# バッジ解除条件に満足度記録を必須化 - 2026年03月14日 08:10

## 作業内容
「初めての休肝日」と「初めての2連続」バッジの解除条件を変更。休肝日を宣言しただけではバッジが解除されず、記録画面で満足度（satisfaction）を入力して保存した時点で初めて解除されるようにした。

## 変更ファイル
- lib/store.ts
- tests/analytics.test.ts
- doc/バッジシステム設計書.md

## 変更詳細

### バッジ解除条件の変更（lib/store.ts）

`checkNewBadges` 関数内の `first_kyukan` と `first_consecutive` の判定条件に `satisfaction !== null` を追加。

#### 変更箇所（行番号）
```typescript
// 変更前（first_kyukan）
if (Object.values(records).some((r) => r.status === "kyukan")) {

// 変更後
if (Object.values(records).some((r) => r.status === "kyukan" && r.satisfaction !== null)) {
```

```typescript
// 変更前（first_consecutive）
records[sortedDates[i - 1]]?.status === "kyukan" &&
records[sortedDates[i]]?.status === "kyukan"

// 変更後
rPrev?.status === "kyukan" && rPrev?.satisfaction !== null &&
rCurr?.status === "kyukan" && rCurr?.satisfaction !== null
```

### テストの追加・修正（tests/analytics.test.ts）

- 満足度なしではバッジが解除されないテストケースを2件追加
- 既存テストのレコードデータに `satisfaction` フィールドを追加

### ドキュメント更新（doc/バッジシステム設計書.md）

解除条件テーブルの `first_kyukan` と `first_consecutive` の説明文を更新。

## 影響範囲
- バッジ解除のタイミングが変わる（宣言時→記録完了時）
- 既にバッジを取得済みのユーザーには影響なし（earned判定でスキップされるため）

## テスト項目
1. 休肝日を宣言しただけではバッジが解除されないこと
2. 休肝日に満足度を記録するとバッジが解除されること
3. 2連続休肝日で片方だけ満足度記録済みの場合はバッジが解除されないこと
4. 2連続休肝日で両日とも満足度記録済みの場合にバッジが解除されること
5. `pnpm test` が全件パスすること ✅
6. `pnpm check` が型エラーなしであること ✅
