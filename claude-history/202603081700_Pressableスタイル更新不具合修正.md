# Pressableのfunction styleによるスタイル更新不具合修正 - 2026年03月08日 17:00

## 作業内容
Pressableのfunction style `({ pressed }) => [...]` 内でstate依存のスタイル（borderColor/backgroundColor）を計算していると、Reanimatedのアニメーションと干渉してスタイルが正しく再評価されないバグを修正。すべての該当箇所をstatic styleに変更した。

## 変更ファイル
- app/(tabs)/record.tsx
- app/declaration.tsx
- app/alternative.tsx

## 変更詳細

### record.tsx: closeDetailOptionsにガード条件追加
`showDetailOptions`が既にfalseの場合、不要な`withTiming(0)`実行をスキップするガード条件を追加。

```typescript
// 変更前
const closeDetailOptions = useCallback(() => {
  setShowDetailOptions(false);
  detailHeight.value = withTiming(0, { duration: 120 });
}, [detailHeight]);

// 変更後
const closeDetailOptions = useCallback(() => {
  if (!showDetailOptions) return;
  setShowDetailOptions(false);
  detailHeight.value = withTiming(0, { duration: 120 });
}, [detailHeight, showDetailOptions]);
```

### record.tsx: 「飲まなかった」ボタンのstatic style化 + closeDetailOptions呼び出し追加
function styleをstatic styleに変更し、onPressでcloseDetailOptions()を呼ぶように修正。

### record.tsx: 杯数ボタン・詳細ボタン・満足度ボタンのstatic style化
DRINK_OPTIONS、DRINK_DETAIL_OPTIONS、SATISFACTION_OPTIONSのPressableをすべてstatic styleに変更。

### declaration.tsx: 上限杯数ボタン・飲みたい理由チップのstatic style化
LIMIT_OPTIONS、REASON_OPTIONSのPressableをstatic styleに変更。

### alternative.tsx: 気分チップ・代替行動カードのstatic style化
MOODSチップ、代替行動カードのPressableをstatic styleに変更。

### 共通: pressed && { opacity } の削除
すべての該当箇所で `pressed && { opacity: 0.8 }` を削除（static styleではpressedが参照できないため）。

## 影響範囲
- 記録画面の杯数選択・満足度選択ボタン
- 飲酒前宣言画面の上限杯数・理由選択ボタン
- 代替行動画面の気分・アクション選択ボタン
- 押下時のopacityフィードバックが削除される（スタイル正確性を優先）
