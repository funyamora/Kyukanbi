## 変更詳細

### toLocalDateStr ヘルパー関数の追加（lib/store.ts）
`toISOString().slice(0, 10)` はUTC基準のため、日本時間の深夜0時〜9時の間に日付がずれる問題があった。
`toLocalDateStr` をエクスポート関数として追加し、`getFullYear()`/`getMonth()`/`getDate()` でローカルタイムゾーンの日付文字列を返すように修正。

以下の箇所で `toISOString().slice(0, 10)` → `toLocalDateStr(d)` に置換:
- `todayStr()`
- `getWeekDates()`
- `weekGoalMet()`
- `computeWeekdayAverages()`
- `getMonthDatesForSummary()`
- `scheduleReminder()`（lib/notifications.ts）
- `shiftDate()`（app/(tabs)/record.tsx）
- `getMonthDates()`（app/(tabs)/review.tsx）
- テストコード（tests/analytics.test.ts）

### 変更箇所（lib/store.ts）
```typescript
// 追加
export function toLocalDateStr(d: Date): string {
  const y = d.getFullYear();
  const m = String(d.getMonth() + 1).padStart(2, "0");
  const day = String(d.getDate()).padStart(2, "0");
  return `${y}-${m}-${day}`;
}

// 変更前
export function todayStr(): string {
  return new Date().toISOString().slice(0, 10);
}

// 変更後
export function todayStr(): string {
  return toLocalDateStr(new Date());
}
```
