# ローカル通知機能の実装 - 2026年03月08日 11:15

## 作業内容
「休肝日つくーる」アプリにローカル通知機能を実装。expo-notificationsを使用して、夜間リマインダーと2連続休肝日達成通知の2種類の通知をサポート。設定画面のUI（AppSettingsの型定義・トグルスイッチ）は既に実装済みだったため、実際の通知スケジューリング・権限管理・達成検知ロジックを追加した。

## 変更詳細

### lib/notifications.ts（新規作成）
通知機能の全関数を集約したモジュール。
- `requestNotificationPermission()` — 権限リクエスト（Android通知チャンネル作成含む）
- `scheduleReminder(time, store)` — DailyTriggerでリマインダーをスケジュール。今日のステータス（kyukan/ok/undecided）に応じて通知内容を変更
- `cancelReminder()` — 固定識別子 `reminder_daily` の通知をキャンセル
- `sendAchievementNotification()` — 即時通知で達成を通知
- `checkConsecutiveAchievement(records, date)` — 指定日を含む週で2連続休肝日が成立しているか判定（テスト対象）

### app/_layout.tsx
- `NotificationSetup` コンポーネントを追加（AppProvider内、return null）
  - 初回起動時に権限リクエスト（AsyncStorageフラグで重複防止）
  - settings/store変更時にリマインダーを再スケジュール
  - 通知タップ時にrouter.pushで画面遷移
- `Notifications.setNotificationHandler` でフォアグラウンド通知表示を設定

### lib/app-context.tsx
- `patchRecord` 内で達成判定を追加
- status が `kyukan` に変更され、achievementNotification が ON かつ 2連続成立時に `sendAchievementNotification()` を呼ぶ

### app/(tabs)/settings.tsx
- 「※ 通知機能は今後のアップデートで実装予定です」の注記テキストを削除（3行）

### tests/notifications.test.ts（新規作成）
- `checkConsecutiveAchievement` の3テストケース
  - 2連続kyukanで true
  - 連続なしで false
  - 1日だけkyukanで false
- react-native / AsyncStorage / expo-notifications をモック

```typescript
const patchRecord = useCallback(
  async (date: string, patch: Partial<DailyRecord>) => {
    const newStore = await updateRecord(date, patch);
    setStore({ ...newStore });
  },

// 変更後
const patchRecord = useCallback(
  async (date: string, patch: Partial<DailyRecord>) => {
    const newStore = await updateRecord(date, patch);
    setStore({ ...newStore });
    if (
      patch.status === "kyukan" &&
      settings.achievementNotification &&
      checkConsecutiveAchievement(newStore.records, date)
    ) {
      sendAchievementNotification();
    }
  },
  [settings.achievementNotification]
);
```
